# Memory Matters More: Event-Centric Memory as a Logic Map for Agent Searching and Reasoning (CompassMem)

- **arXiv id:** 2601.04726
- **Date:** 2026-01
- **URL:** https://www.arxiv.org/abs/2601.04726
- **Taxonomy tags:** Factual / Token-level / Retrieval
- **Authors:** Hu, Liu, Tan, Zhu, Dou

## Problem

Most agent memory is stored flatly (independent text snippets) and
retrieved by similarity alone. Even memory systems that add *some*
structure rarely capture explicit logical relations — causality, temporal
order — between memory units, so agents can't reason across long-horizon
dependencies; they can only find things that look similar to the query.

## Mechanism

CompassMem is inspired by Event Segmentation Theory (how humans naturally
chunk continuous experience into discrete events). It incrementally builds
an **Event Graph**: nodes are coherent event units produced by segmenting
experience, edges encode explicit logical relations (causal, temporal).
At inference time, the agent treats this graph as a "logic map" and
performs **structured, goal-directed navigation** over it — following
logical edges rather than only ranking by embedding distance. This
consistently improves both retrieval and reasoning on LoCoMo and
NarrativeQA, especially multi-hop and temporal-reasoning questions.

## What's reusable for corticore

The core insight — that *relations between memories* (not just their
individual content) carry retrieval-relevant information — is real and not
addressed at all by corticore's v0.1 `dynamics/retrieval.py`, which scores
every memory independently. A cheap, non-graph-database version of this
idea is plausible for a future round: store lightweight `causal`/`temporal`
edges as rows in whichever `MemoryStore` is active (SQLite or the new
`PostgresStore`), without needing a graph database — this is a candidate
seam, not implemented this round.

## Setup cost

Full adoption requires an event-segmentation model/pipeline and a relation
extractor, which is nontrivial NLP work, not just infrastructure. Even a
scaled-down "explicit edges" version adds real design surface (traversal
strategy, edge-decay semantics) that deserves its own ADR when it's
actually built.
