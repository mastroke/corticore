# Agentic Memory: Learning Unified Long-Term and Short-Term Memory Management for Large Language Model Agents (AgeMem)

- **arXiv id:** 2601.01885
- **Date:** 2026-01
- **URL:** https://arxiv.org/abs/2601.01885
- **Taxonomy tags:** Working memory / Token-level / Formation + Evolution
- **Authors:** Yu, Yao, Xie, Tan, Feng, Li, Wu (Alibaba Group, Wuhan University)

## Problem

LLM agents split long-term memory (LTM) and short-term memory (STM) into
separate, independently-optimized components — STM is patched with RAG or
periodic summarization on a fixed schedule; LTM is managed by handcrafted
trigger rules or a bolted-on "memory manager" model. Because the two are
combined ad hoc rather than jointly, agents miss infrequent-but-critical
details, accumulate noise, and can't end-to-end optimize *when* and *what*
to remember versus forget.

## Mechanism

AgeMem exposes memory operations (store, retrieve, update, summarize,
discard) as **tool-based actions inside the agent's own policy**, so a single
model decides, in context, what belongs in LTM vs. STM and when to act. It's
trained with a three-stage progressive reinforcement learning strategy and a
step-wise GRPO variant designed for the sparse, discontinuous rewards that
memory operations produce (you don't know if a "store" was worth it until
much later in the episode). Evaluated across five long-horizon benchmarks
against memory-augmented baselines, on multiple LLM backbones.

## What's reusable for corticore

corticore is a library, not an agent policy — we don't train a model to
decide store/discard timing, and doing so would break the zero-setup
promise (it requires RL training infra). But two ideas transfer directly:

1. **Memory operations as a small, explicit action surface**
   (store/retrieve/update/summarize/discard) is exactly the shape of our
   public API (`remember`, `recall`, `reflect` covers update/summarize/discard
   as one consolidated pass). AgeMem validates that this five-verb surface is
   sufficient — we don't need more.
2. **Reward is sparse and discontinuous for memory decisions** — i.e. you
   can't score a "should I have forgotten this?" decision at the moment it
   happens. This is a caution against corticore's `reflect()` ever trying to
   be "smart" about long-horizon task reward; it should stay a legible,
   inspectable heuristic (decay + similarity), not a black-box policy. This
   directly reinforces the `why()` differentiator.

## Setup cost

High if adopted literally (RL training pipeline, policy fine-tuning, GPU).
Not compatible with the zero-setup default path. Any future integration
would have to live behind an optional extra that calls an external policy,
never in the default `LocalEmbedder`/`SQLiteStore` path.
